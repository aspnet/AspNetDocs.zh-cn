## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="b688b-111">部署成员资格数据库时的问题是什么？</span><span class="sxs-lookup"><span data-stu-id="b688b-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="b688b-112">在大多数情况下，设计数据库的部署策略时，需要考虑的第一件事是要部署的数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="b688b-113">在开发或测试环境中，你可能想要部署用户帐户数据以便快速轻松地进行测试。</span><span class="sxs-lookup"><span data-stu-id="b688b-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="b688b-114">在过渡环境或生产环境中，您不太可能需要部署用户帐户数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="b688b-115">遗憾的是，ASP.NET 的成员资格数据库引入了一些具体的难题，这使得这一决定变得更为复杂：</span><span class="sxs-lookup"><span data-stu-id="b688b-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="b688b-116">仅限架构的部署会使成员资格数据库处于不可操作状态。</span><span class="sxs-lookup"><span data-stu-id="b688b-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="b688b-117">这是因为成员资格数据库包含数据库在运行时所需的一些配置数据（在**aspnet\_SchemaVersions**表中）。</span><span class="sxs-lookup"><span data-stu-id="b688b-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="b688b-118">因此，如果您对成员资格数据库执行仅限架构的部署以排除用户帐户数据，则需要运行部署后脚本来添加必要的配置数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="b688b-119">根据成员资格数据库的配置方式，成员资格提供程序可以使用计算机密钥来加密密码，并将其存储在数据库中。</span><span class="sxs-lookup"><span data-stu-id="b688b-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="b688b-120">在这种情况下，使用数据库部署的任何用户帐户数据在目标服务器上将变为不可用。</span><span class="sxs-lookup"><span data-stu-id="b688b-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="b688b-121">出于此原因，不支持部署用户帐户数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="b688b-122">选择成员资格数据库策略</span><span class="sxs-lookup"><span data-stu-id="b688b-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="b688b-123">当你选择如何在企业服务器环境中设置成员资格数据库时，请使用以下准则：</span><span class="sxs-lookup"><span data-stu-id="b688b-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="b688b-124">如果可能，请不要部署成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="b688b-125">而应在目标数据库服务器上手动创建成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="b688b-126">如果尚未自定义成员资格数据库架构，只需使用[ASP.NET SQL Server 注册工具（aspnet\_regsql）](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx)在目标的原位中创建一个新的架构。</span><span class="sxs-lookup"><span data-stu-id="b688b-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="b688b-127">例如，如果您没有选项，而是部署成员&#x2014;资格数据库（例如，如果您对数据库架构&#x2014;进行了大量修改，则应执行成员资格数据库的仅限架构的部署，排除用户帐户数据，然后运行部署后脚本以添加任何所需的配置数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="b688b-128">您可以在[如何：部署 ASP.NET 成员资格数据库，而无需包含用户帐户](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx)中找到有关这些方法的广泛指导。</span><span class="sxs-lookup"><span data-stu-id="b688b-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="b688b-129">请务必记住，*成员资格数据库的架构可能是相当静态*的。</span><span class="sxs-lookup"><span data-stu-id="b688b-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="b688b-130">即使您已自定义成员资格数据库，也不太可能需要定期更新架构，&#x2014;其频率与 web 应用程序或数据库项目中的代码的频率相同。</span><span class="sxs-lookup"><span data-stu-id="b688b-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="b688b-131">因此，您不需要在任何自动或单步部署过程中都包含成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="b688b-132">使用 VSDBCMD 更新成员资格数据库架构</span><span class="sxs-lookup"><span data-stu-id="b688b-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="b688b-133">如果在第一次部署后修改成员资格数据库的结构，则可能不希望使用 Internet Information Services （IIS） Web 部署工具（Web 部署）来重新部署数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="b688b-134">Web 部署中的数据库部署功能不包括将差异更新到目标数据库&#x2014;的功能，Web 部署必须删除并重新创建数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="b688b-135">这意味着会丢失任何现有的用户帐户数据，这通常不需要过渡或生产环境。</span><span class="sxs-lookup"><span data-stu-id="b688b-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="b688b-136">替代方法是使用 VSDBCMD 实用工具更新目标数据库的架构。</span><span class="sxs-lookup"><span data-stu-id="b688b-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="b688b-137">VSDBCMD 包括两项重要功能。</span><span class="sxs-lookup"><span data-stu-id="b688b-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="b688b-138">首先，它可以将现有数据库的架构导入到 .dbschema 文件中。</span><span class="sxs-lookup"><span data-stu-id="b688b-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="b688b-139">其次，它可以将 .dbschema 文件作为差异更新部署到现有数据库，这意味着它只会使目标数据库保持最新状态，而不会丢失任何数据。</span><span class="sxs-lookup"><span data-stu-id="b688b-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

<span data-ttu-id="b688b-140">您可以使用这些高级步骤更新成员资格数据库架构：</span><span class="sxs-lookup"><span data-stu-id="b688b-140">You can use these high-level steps to update a membership database schema:</span></span>

1. <span data-ttu-id="b688b-141">使用 VSDBCMD**导入**操作为源成员资格数据库生成 .dbschema 文件。</span><span class="sxs-lookup"><span data-stu-id="b688b-141">Use the VSDBCMD **Import** action to generate a .dbschema file for your source membership database.</span></span> <span data-ttu-id="b688b-142">[如何：从命令提示符导入架构](https://msdn.microsoft.com/library/dd172135.aspx)中介绍了此过程。</span><span class="sxs-lookup"><span data-stu-id="b688b-142">This procedure is described in [How to: Import a Schema from a Command Prompt](https://msdn.microsoft.com/library/dd172135.aspx).</span></span>
2. <span data-ttu-id="b688b-143">使用 VSDBCMD**部署**操作将 .dbschema 文件部署到目标成员身份数据库。</span><span class="sxs-lookup"><span data-stu-id="b688b-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="b688b-144">[VSDBCMD 的命令行参考中介绍了此过程。EXE （部署和架构导入）](https://msdn.microsoft.com/library/dd193283.aspx)。</span><span class="sxs-lookup"><span data-stu-id="b688b-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
