## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="6cae7-111">在将成员资格数据库时，问题是什么？</span><span class="sxs-lookup"><span data-stu-id="6cae7-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="6cae7-112">在大多数情况下，当数据库中，通过部署策略时需要考虑的第一件事是你想要部署哪些的数据。</span><span class="sxs-lookup"><span data-stu-id="6cae7-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="6cae7-113">在开发或测试环境中，你可能想要将用户帐户数据，以便快速且轻松地测试部署。</span><span class="sxs-lookup"><span data-stu-id="6cae7-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="6cae7-114">在过渡或生产环境中，是不太可能想要将用户帐户数据部署。</span><span class="sxs-lookup"><span data-stu-id="6cae7-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="6cae7-115">遗憾的是，ASP.NET 成员资格数据库引入做出这一决定更复杂的一些特定挑战：</span><span class="sxs-lookup"><span data-stu-id="6cae7-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="6cae7-116">仅限架构的部署将使处于非正常运行状态的成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="6cae7-117">这是因为成员资格数据库包含某些配置数据 (在**aspnet\_SchemaVersions**表) 的数据库正常运行。</span><span class="sxs-lookup"><span data-stu-id="6cae7-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="6cae7-118">在这种情况下，如果以排除用户帐户数据执行成员资格数据库的仅限架构的部署，你将需要运行后期部署脚本添加基本配置数据。</span><span class="sxs-lookup"><span data-stu-id="6cae7-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="6cae7-119">根据成员资格数据库的配置方式，成员资格提供程序可以使用计算机密钥对密码进行加密并将其存储在数据库中。</span><span class="sxs-lookup"><span data-stu-id="6cae7-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="6cae7-120">在这种情况下，与数据库部署的任何用户帐户数据将成为目标服务器上不可用。</span><span class="sxs-lookup"><span data-stu-id="6cae7-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="6cae7-121">出于此原因，用户帐户数据部署不受支持的方案。</span><span class="sxs-lookup"><span data-stu-id="6cae7-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="6cae7-122">选择成员资格数据库策略</span><span class="sxs-lookup"><span data-stu-id="6cae7-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="6cae7-123">选择如何预配企业服务器环境中的成员资格数据库时，请使用以下准则：</span><span class="sxs-lookup"><span data-stu-id="6cae7-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="6cae7-124">如有可能，请不要部署成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="6cae7-125">相反，在目标数据库服务器上手动创建成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="6cae7-126">如果你未自定义成员资格数据库架构，您可以只需创建一个新就地在目标中使用[ASP.NET SQL Server 注册工具 (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx)。</span><span class="sxs-lookup"><span data-stu-id="6cae7-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="6cae7-127">如果别无选择，但若要部署成员资格数据库&#x2014;例如，如果进行了大量修改对数据库架构&#x2014;您应该使用成员资格数据库，若要排除的用户帐户数据的仅限架构的部署，然后运行后期部署脚本来添加任何所需的配置数据。</span><span class="sxs-lookup"><span data-stu-id="6cae7-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="6cae7-128">您可以在这些方法中找到全面的指南[如何：将 ASP.NET 成员资格数据库部署不包括用户帐户的情况下](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx)。</span><span class="sxs-lookup"><span data-stu-id="6cae7-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="6cae7-129">务必要记住*的成员资格数据库架构很可能是较为静态*。</span><span class="sxs-lookup"><span data-stu-id="6cae7-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="6cae7-130">即使你已自定义成员资格数据库，也是不太可能需要更新定期架构&#x2014;不会更改 web 应用程序或数据库项目中的代码的频率相同。</span><span class="sxs-lookup"><span data-stu-id="6cae7-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="6cae7-131">在这种情况下，您无需在任何自动化或单步执行部署过程中包括的成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="6cae7-132">使用 VSDBCMD 更新成员资格数据库架构</span><span class="sxs-lookup"><span data-stu-id="6cae7-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="6cae7-133">如果在第一次部署后修改的成员资格数据库结构，可能不想要使用 Internet 信息服务 (IIS) Web 部署工具 （Web 部署） 来重新部署该数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="6cae7-134">Web 部署中的数据库部署功能不包括目标数据库进行差异更新的功能&#x2014;相反，Web 部署必须删除并重新创建数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="6cae7-135">这意味着，您会丢失在过渡或生产环境中通常不需要任何现有用户帐户数据。</span><span class="sxs-lookup"><span data-stu-id="6cae7-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="6cae7-136">替代方法是使用 VSDBCMD 实用程序来更新您的目标数据库的架构。</span><span class="sxs-lookup"><span data-stu-id="6cae7-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="6cae7-137">VSDBCMD 包括两个重要功能。</span><span class="sxs-lookup"><span data-stu-id="6cae7-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="6cae7-138">首先，它可以导入.dbschema 文件的现有数据库的架构。</span><span class="sxs-lookup"><span data-stu-id="6cae7-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="6cae7-139">其次，它可以作为差异更新，这意味着它仅可以将最新的目标数据库所需的更改，并且不会丢失任何数据，对现有数据库中部署.dbschema 文件。</span><span class="sxs-lookup"><span data-stu-id="6cae7-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

<span data-ttu-id="6cae7-140">您可以使用以下高级步骤更新成员资格数据库架构：</span><span class="sxs-lookup"><span data-stu-id="6cae7-140">You can use these high-level steps to update a membership database schema:</span></span>

1. <span data-ttu-id="6cae7-141">使用 VSDBCMD**导入**操作生成.dbschema 文件源成员资格数据库。</span><span class="sxs-lookup"><span data-stu-id="6cae7-141">Use the VSDBCMD **Import** action to generate a .dbschema file for your source membership database.</span></span> <span data-ttu-id="6cae7-142">此过程所述[如何：从命令提示符下导入架构](https://msdn.microsoft.com/library/dd172135.aspx)。</span><span class="sxs-lookup"><span data-stu-id="6cae7-142">This procedure is described in [How to: Import a Schema from a Command Prompt](https://msdn.microsoft.com/library/dd172135.aspx).</span></span>
2. <span data-ttu-id="6cae7-143">使用 VSDBCMD**部署**.dbschema 文件部署到您的目标成员资格数据库的操作。</span><span class="sxs-lookup"><span data-stu-id="6cae7-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="6cae7-144">此过程所述[VSDBCMD 的命令行参考。EXE （部署和架构导入）](https://msdn.microsoft.com/library/dd193283.aspx)。</span><span class="sxs-lookup"><span data-stu-id="6cae7-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
