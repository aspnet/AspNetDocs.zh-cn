## <a name="pre-generating-views"></a><span data-ttu-id="78472-255">预生成视图</span><span class="sxs-lookup"><span data-stu-id="78472-255">Pre-Generating Views</span></span>

<span data-ttu-id="78472-256">第一次在新的应用程序域中创建 `ObjectContext` 对象时，实体框架将生成一组用于访问数据库的类。</span><span class="sxs-lookup"><span data-stu-id="78472-256">When an `ObjectContext` object is first created in a new application domain, the Entity Framework generates a set of classes that it uses to access the database.</span></span> <span data-ttu-id="78472-257">这些类称为*视图*，如果你有一个非常大的数据模型，则生成这些视图会在新的应用程序域初始化之后延迟网站对第一个页面请求的响应。</span><span class="sxs-lookup"><span data-stu-id="78472-257">These classes are called *views*, and if you have a very large data model, generating these views can delay the web site's response to the first request for a page after a new application domain is initialized.</span></span> <span data-ttu-id="78472-258">可以通过在编译时（而不是在运行时）创建视图来减少此第一次请求延迟。</span><span class="sxs-lookup"><span data-stu-id="78472-258">You can reduce this first-request delay by creating the views at compile time rather than at run time.</span></span>
