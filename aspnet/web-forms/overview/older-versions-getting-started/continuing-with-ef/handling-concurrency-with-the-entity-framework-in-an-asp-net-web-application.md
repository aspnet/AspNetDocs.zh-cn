## <a name="concurrency-conflicts"></a><span data-ttu-id="122c9-116">并发冲突</span><span class="sxs-lookup"><span data-stu-id="122c9-116">Concurrency Conflicts</span></span>

### <a name="pessimistic-concurrency-locking"></a><span data-ttu-id="122c9-121">悲观并发（锁定）</span><span class="sxs-lookup"><span data-stu-id="122c9-121">Pessimistic Concurrency (Locking)</span></span>

<span data-ttu-id="122c9-127">管理锁有一些缺点。</span><span class="sxs-lookup"><span data-stu-id="122c9-127">Managing locks has some disadvantages.</span></span> <span data-ttu-id="122c9-128">编程可能很复杂。</span><span class="sxs-lookup"><span data-stu-id="122c9-128">It can be complex to program.</span></span> <span data-ttu-id="122c9-129">它需要大量的数据库管理资源，如果应用程序的用户数增加（也就是说，它不能很好地进行缩放），则可能会导致性能问题。</span><span class="sxs-lookup"><span data-stu-id="122c9-129">It requires significant database management resources, and it can cause performance problems as the number of users of an application increases (that is, it doesn't scale well).</span></span> <span data-ttu-id="122c9-130">由于这些原因，并不是所有的数据库管理系统都支持悲观并发。</span><span class="sxs-lookup"><span data-stu-id="122c9-130">For these reasons, not all database management systems support pessimistic concurrency.</span></span> <span data-ttu-id="122c9-131">实体框架不提供对它的内置支持，本教程不会演示如何实现它。</span><span class="sxs-lookup"><span data-stu-id="122c9-131">The Entity Framework provides no built-in support for it, and this tutorial doesn't show you how to implement it.</span></span>
