### <a name="how-the-membership-framework-handles-invalid-login-attempts"></a><span data-ttu-id="287f1-150">成员资格框架如何处理无效的登录尝试</span><span class="sxs-lookup"><span data-stu-id="287f1-150">How the Membership Framework Handles Invalid Login Attempts</span></span>

<span data-ttu-id="287f1-151">当访问者到达登录页面并提交其凭据时，他们的浏览器会向登录页发出 HTTP 请求。</span><span class="sxs-lookup"><span data-stu-id="287f1-151">When a visitor reaches the login page and submits their credentials, their browser makes an HTTP request to the login page.</span></span> <span data-ttu-id="287f1-152">如果凭据有效，HTTP 响应将在 cookie 中包含身份验证票证。</span><span class="sxs-lookup"><span data-stu-id="287f1-152">If the credentials are valid, the HTTP response includes the authentication ticket in a cookie.</span></span> <span data-ttu-id="287f1-153">因此，尝试侵入网站的黑客可能会创建一个程序，该程序使用有效的用户名并在密码上猜测来向登录页发送 HTTP 请求。</span><span class="sxs-lookup"><span data-stu-id="287f1-153">Therefore, a hacker attempting to break into your site could create a program that exhaustively sends HTTP requests to the login page with a valid username and a guess at the password.</span></span> <span data-ttu-id="287f1-154">如果密码推测正确，则登录页将返回身份验证票证 cookie，此时程序知道该 cookie 已跌倒有效的用户名/密码对。</span><span class="sxs-lookup"><span data-stu-id="287f1-154">If the password guess is correct, the login page will return the authentication ticket cookie, at which point the program knows it has stumbled upon a valid username/password pair.</span></span> <span data-ttu-id="287f1-155">通过暴力破解，此类程序可能能够碰到用户密码，尤其是在密码较弱的情况下。</span><span class="sxs-lookup"><span data-stu-id="287f1-155">Through brute force, such a program might be able to stumble upon a user's password, especially if the password is weak.</span></span>

<span data-ttu-id="287f1-156">若要防止这种暴力攻击，成员身份框架会锁定用户，前提是在某个时间段内有一定次数的失败登录尝试。</span><span class="sxs-lookup"><span data-stu-id="287f1-156">To prevent such brute force attacks, the Membership framework locks out a user if there are a certain number of unsuccessful login attempts within a certain period of time.</span></span> <span data-ttu-id="287f1-157">可以通过以下两个成员资格提供程序配置设置来配置确切的参数：</span><span class="sxs-lookup"><span data-stu-id="287f1-157">The exact parameters are configurable through the following two Membership provider configuration settings:</span></span>

- <span data-ttu-id="287f1-158">`maxInvalidPasswordAttempts` -指定在帐户被锁定之前的时间段内允许用户进行的无效密码尝试次数。默认值为5。</span><span class="sxs-lookup"><span data-stu-id="287f1-158">`maxInvalidPasswordAttempts` - specifies how many invalid password attempts are allowed for the user within the time period before the account is locked out. The default value is 5.</span></span>
