## <a name="reduced-latency"></a><span data-ttu-id="6111c-127">减少延迟</span><span class="sxs-lookup"><span data-stu-id="6111c-127">Reduced Latency</span></span>

<span data-ttu-id="6111c-128">每次执行耗时的工作时，队列都非常有用。</span><span class="sxs-lookup"><span data-stu-id="6111c-128">Queues are useful any time you are doing time-consuming work.</span></span> <span data-ttu-id="6111c-129">如果任务花费几秒钟或更长时间，而不是阻止最终用户，请将工作项放入队列中。</span><span class="sxs-lookup"><span data-stu-id="6111c-129">If a task takes a few seconds or longer, instead of blocking the end user, put the work item into a queue.</span></span> <span data-ttu-id="6111c-130">告诉用户 "我们正在处理该用户"，然后使用队列侦听器在后台处理该任务。</span><span class="sxs-lookup"><span data-stu-id="6111c-130">Tell the user "We're working on it," and then use a queue listener to process the task in the background.</span></span>

## <a name="rate-leveling-and-independent-scaling"></a><span data-ttu-id="6111c-159">分级调配和独立缩放</span><span class="sxs-lookup"><span data-stu-id="6111c-159">Rate Leveling and Independent Scaling</span></span>

<span data-ttu-id="6111c-160">队列还适用于称为*速率调节*或*负载分级*的内容。</span><span class="sxs-lookup"><span data-stu-id="6111c-160">Queues are also useful for something called *rate leveling* or *load leveling*.</span></span>

<span data-ttu-id="6111c-161">Web 应用通常容易遭受流量突发。</span><span class="sxs-lookup"><span data-stu-id="6111c-161">Web apps are often susceptible to sudden bursts in traffic.</span></span> <span data-ttu-id="6111c-162">尽管可以使用自动缩放来自动添加 web 服务器来处理增加的 web 流量，但自动缩放可能无法快速响应，以致无法处理负载突然高峰。</span><span class="sxs-lookup"><span data-stu-id="6111c-162">While you can use autoscaling to automatically add web servers to handle increased web traffic, autoscaling might not be able to react quickly enough to handle abrupt spikes in load.</span></span> <span data-ttu-id="6111c-163">如果 web 服务器可以通过将消息写入队列来卸载其中的某些工作，则它们可以处理更多流量。</span><span class="sxs-lookup"><span data-stu-id="6111c-163">If the web servers can offload some of the work they have to do by writing a message to a queue, they can handle more traffic.</span></span> <span data-ttu-id="6111c-164">然后，后端服务可以从队列中读取消息并对其进行处理。</span><span class="sxs-lookup"><span data-stu-id="6111c-164">A backend service can then read messages from the queue and process them.</span></span> <span data-ttu-id="6111c-165">当传入负载变化时，队列的深度将增大或缩小。</span><span class="sxs-lookup"><span data-stu-id="6111c-165">The depth of the queue will grow or shrink as the incoming load varies.</span></span>
