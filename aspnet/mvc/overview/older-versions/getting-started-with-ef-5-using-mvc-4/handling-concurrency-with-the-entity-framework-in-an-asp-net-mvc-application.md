<span data-ttu-id="7af43-119">当某用户显示实体数据以对其进行编辑，而另一用户在上一用户的更改写入数据库之前更新同一实体的数据时，会发生并发冲突。</span><span class="sxs-lookup"><span data-stu-id="7af43-119">A concurrency conflict occurs when one user displays an entity's data in order to edit it, and then another user updates the same entity's data before the first user's change is written to the database.</span></span> <span data-ttu-id="7af43-120">如果不启用此类冲突的检测，则最后更新数据库的人员将覆盖其他用户的更改。</span><span class="sxs-lookup"><span data-stu-id="7af43-120">If you don't enable the detection of such conflicts, whoever updates the database last overwrites the other user's changes.</span></span> <span data-ttu-id="7af43-121">在许多应用程序中，此风险是可接受的：如果用户很少或更新很少，或者一些更改被覆盖并不重要，则并发编程可能弊大于利。</span><span class="sxs-lookup"><span data-stu-id="7af43-121">In many applications, this risk is acceptable: if there are few users, or few updates, or if isn't really critical if some changes are overwritten, the cost of programming for concurrency might outweigh the benefit.</span></span> <span data-ttu-id="7af43-122">在此情况下，不必配置应用程序来处理并发冲突。</span><span class="sxs-lookup"><span data-stu-id="7af43-122">In that case, you don't have to configure the application to handle concurrency conflicts.</span></span>

### <a name="pessimistic-concurrency-locking"></a><span data-ttu-id="7af43-123">悲观并发 （锁定）</span><span class="sxs-lookup"><span data-stu-id="7af43-123">Pessimistic Concurrency (Locking)</span></span>

<span data-ttu-id="7af43-129">管理锁定有缺点。</span><span class="sxs-lookup"><span data-stu-id="7af43-129">Managing locks has disadvantages.</span></span> <span data-ttu-id="7af43-130">编程可能很复杂。</span><span class="sxs-lookup"><span data-stu-id="7af43-130">It can be complex to program.</span></span> <span data-ttu-id="7af43-131">它需要大量的数据库管理资源，并且它可能导致性能问题的应用程序的用户数增加 （即，它不能很好）。</span><span class="sxs-lookup"><span data-stu-id="7af43-131">It requires significant database management resources, and it can cause performance problems as the number of users of an application increases (that is, it doesn't scale well).</span></span> <span data-ttu-id="7af43-132">由于这些原因，并不是所有的数据库管理系统都支持悲观并发。</span><span class="sxs-lookup"><span data-stu-id="7af43-132">For these reasons, not all database management systems support pessimistic concurrency.</span></span> <span data-ttu-id="7af43-133">实体框架不提供内置支持，以及本教程不展示如何实现它。</span><span class="sxs-lookup"><span data-stu-id="7af43-133">The Entity Framework provides no built-in support for it, and this tutorial doesn't show you how to implement it.</span></span>
