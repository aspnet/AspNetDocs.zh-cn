## <a name="conventions"></a><span data-ttu-id="27acc-314">约定</span><span class="sxs-lookup"><span data-stu-id="27acc-314">Conventions</span></span>

<span data-ttu-id="27acc-315">您必须在实体框架能够为您创建完整的数据库中编写的代码量是由于最小*约定*，或使实体框架的假设。</span><span class="sxs-lookup"><span data-stu-id="27acc-315">The amount of code you had to write in order for Entity Framework to be able to create a complete database for you is minimal because of *conventions*, or assumptions that Entity Framework makes.</span></span> <span data-ttu-id="27acc-316">其中一些已记下或使用而无需你不知道的：</span><span class="sxs-lookup"><span data-stu-id="27acc-316">Some of them have already been noted or were used without your being aware of them:</span></span>

- <span data-ttu-id="27acc-317">复数的形式的实体类名称用作表名。</span><span class="sxs-lookup"><span data-stu-id="27acc-317">The pluralized forms of entity class names are used as table names.</span></span>
- <span data-ttu-id="27acc-318">使用实体属性名作为列名。</span><span class="sxs-lookup"><span data-stu-id="27acc-318">Entity property names are used for column names.</span></span>
- <span data-ttu-id="27acc-319">命名的实体属性`ID`或*classname* `ID`被视为主键属性。</span><span class="sxs-lookup"><span data-stu-id="27acc-319">Entity properties that are named `ID` or *classname* `ID` are recognized as primary key properties.</span></span>
- <span data-ttu-id="27acc-320">如果它名为一个属性将被解释为外键属性 *&lt;导航属性名称&gt;&lt;主键属性名称&gt;* (例如， `StudentID` 为`Student`导航属性所以`Student`实体的主键是`ID`)。</span><span class="sxs-lookup"><span data-stu-id="27acc-320">A property is interpreted as a foreign key property if it's named *&lt;navigation property name&gt;&lt;primary key property name&gt;* (for example, `StudentID` for the `Student` navigation property since the `Student` entity's primary key is `ID`).</span></span> <span data-ttu-id="27acc-321">外键属性还可以进行命名相同只需&lt;主键属性名称&gt;(例如，`EnrollmentID`由于`Enrollment`实体的主键是`EnrollmentID`)。</span><span class="sxs-lookup"><span data-stu-id="27acc-321">Foreign key properties can also be named the same simply &lt;primary key property name&gt; (for example, `EnrollmentID` since the `Enrollment` entity's primary key is `EnrollmentID`).</span></span>

<span data-ttu-id="27acc-322">您已了解约定可以重写。</span><span class="sxs-lookup"><span data-stu-id="27acc-322">You've seen that conventions can be overridden.</span></span> <span data-ttu-id="27acc-323">例如，指定表名不应变为复数形式，并稍后您将看到如何显式标记为外键属性的属性。</span><span class="sxs-lookup"><span data-stu-id="27acc-323">For example, you specified that table names shouldn't be pluralized, and you'll see later how to explicitly mark a property as a foreign key property.</span></span>
