<span data-ttu-id="a9bb2-118">公共语言运行时 (CLR) 需要重载方法拥有唯一的参数签名（相同的方法名称但不同的参数列表）。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-118">The common language runtime (CLR) requires overloaded methods to have a unique parameter signature (same method name but different list of parameters).</span></span> <span data-ttu-id="a9bb2-119">但是，这里需要两个 `Delete` 方法 -- 一个用于 GET，另一个用于 POST -- 这两个方法拥有相同的参数签名。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-119">However, here you need two `Delete` methods -- one for GET and one for POST -- that both have the same parameter signature.</span></span> <span data-ttu-id="a9bb2-120">（它们都需要接受单个整数作为参数。）</span><span class="sxs-lookup"><span data-stu-id="a9bb2-120">(They both need to accept a single integer as a parameter.)</span></span>

<span data-ttu-id="a9bb2-121">可通过两种方法解决此问题，一种是为方法提供不同的名称。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-121">There are two approaches to this problem, one is to give the methods different names.</span></span> <span data-ttu-id="a9bb2-122">这正是前面的示例中的基架机制进行的操作。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-122">That's what the scaffolding mechanism did in the preceding example.</span></span> <span data-ttu-id="a9bb2-123">但是，这会造成一个小问题：ASP.NET 按名称将 URL 段映射到操作方法，如果重命名方法，则路由通常无法找到该方法。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-123">However, this introduces a small problem: ASP.NET maps segments of a URL to action methods by name, and if you rename a method, routing normally wouldn't be able to find that method.</span></span> <span data-ttu-id="a9bb2-124">该示例中也提供了解决方案，即向 `DeleteConfirmed` 方法添加 `ActionName("Delete")` 属性。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-124">The solution is what you see in the example, which is to add the `ActionName("Delete")` attribute to the `DeleteConfirmed` method.</span></span> <span data-ttu-id="a9bb2-125">该属性对路由系统执行映射，以便包括 POST 请求的 /Delete/ 的 URL可找到 `DeleteConfirmed` 方法。</span><span class="sxs-lookup"><span data-stu-id="a9bb2-125">That attribute performs mapping for the routing system so that a URL that includes /Delete/ for a POST request will find the `DeleteConfirmed` method.</span></span>
